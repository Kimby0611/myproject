1. 프로젝트 개요
		
	주제 : 회사 자산관리 사이트
		회사 실물 자산 관리 프로그램(부서코드, 자산번호, 물품명 등) 추가 조회 삭제 기능 관리자 만들어서 권한부여
		추가구현 >> 회사 홈페이지로 봐도 무방 회사 홈페이지 만들고 관리자 권한이 부여된 유저만 자산관리 사이트에 접속 가능, 회사 위치 지도, 날씨
		 
			

	목표 : 간단한 기능들로 전체적인 웹 사이트 제작하여 흐름 익히기, ai로 직접적인 코드를 받지 않고 만들어 보기.
		   react로 웹 페이지 제작 후 springBoot로 백앤드 구상 후 MariaDB로 연동하기
		   가능하다면 BootStrap사용해 보기 (단순 연습용)
		   사용자별로 권한을 나누어서 추가, 삭제, 검색 등의 기능을 권한별로 나누고, 사용자 시점의 ui/ux 만들기

3. 시나리오 설계
	사용자 시나리오
		- 관리자 : 회사 내의 모든 사용자에게 권한을 부여할 수 있고, 자산 수정 가능
		- 중간 관리자(부서장) : 해당 부서 내의 사용자의 자산 수정 및 팀원에게 일시적 권한 부여 가능
		- 일반 사용자 : 자산 검색만 가능
		- 비로그인 사용자(추가 사항) : 홈페이지 구경만 가능 자산 보기 불가능
		- 메인 페이지 필터 : 관리자, 부서, 가격, 이름, 코드, 위치?
		필요한거 있으면 추가해주세요
		
	프로토타입 설계
		- UI : 종이에 스케치 후 구현
		- ERD설계 : starUML로 DB 스키마 설계
	
	DB 구축
		- JAP로 MariaDB 테이블 생성
		- 테스트 데이터 생성 후 삽입
		- DBeaver로 데이터 확인
		
	프론트엔드 - 백엔드 연동 및 API테스트
		- Spring Boot의 엔드포인트로 등록 구현 후 테스트
		
	
	프론트엔드 구현
		-CSS/bootstarp으로 기본 레이아웃 구성
		-로그인등 입력 폼 구현
		-Axios로 백앤드 호출
	
	백엔드 구현
		-Spring Security로 인증 관리
		-REST API 구현
		-MariaDB와 JPA로 연동
	
	오류사항 확인
		필요한거 적기
 
	예외 처리
		필요한거 적기
	
4. 개발 우선순위

	1. ERD 설계, DB구축 및, 프론트 스케치 및 프로토타입 제작(간단한 페이지와 버튼만)
	2. react와 springBoot 연동 및 회원가입과 로그인 기능 구현, react와 springBoot 연동 및 JPA로 DB에 테스트 데이터 삽입 후 테스트
	3. 관리자 페이지 구현, 메인 페이지 기능구현
	4. UI 디자인 마무리, 예외 및 보안처리
	5. 이후 추가 >> 날씨, 지도 등
	
	
5. 사이트 흐름
	localhost:3000/login(로그인) >> localhost:3000/home(홈) >> localhost:3000/checkpassword(패스워드 확인) >> localhost:3000/user(비밀번호 변경 및 개인정보 핸들링)
														   >> admin-dialog(관리자가 재고수정)
														   >> localhost:3000/admin(관리자의 직원관리)
														   
														   
6. 추가사항
	- 필터의 수를 줄이고 메뉴 네비바를 만들어 전체와 그룹별로 묶어서 보여준 후 필터기능 추가(드롭다운 x)
		ex) 메뉴(전체, 전자기기, 비전자기기, 기타) 필터 (가격, 관리자, 부서)
	- 첫 화면에 전체를 보여준 후 기존의 필터 외의 드롭다운 메뉴 네비바 생성(드롭다운 o)
		ex) 메뉴(전체, 전자기기(컴퓨터, 모니터, 키보드), 비전자기기(사물함, 책상, 의자), 기타(?)) 
	- 하나의 사이트로 운영하는 방법
		마이페이지를 확장시켜 비밀번호 변경 외의 프로필, 회사 내의 직무, 성과 등 추가
		지도와 날씨 추가
